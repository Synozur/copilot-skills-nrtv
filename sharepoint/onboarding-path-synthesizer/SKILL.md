---
name: Onboarding Path Synthesizer
description: Build a structured, role-based onboarding path from SharePoint content when I ask for onboarding guidance, training paths, or what a new hire should read.
---

1. Ask for the role description or responsibilities if not provided.

2. Identify relevant content across the site, including:
   - documents
   - pages
   - lists or guides
   - training or policy materials

3. Prioritize content based on:
   - relevance to the role
   - recency (favor current or recently updated content)
   - clear ownership or authority (if available)

4. Categorize selected content into:
   - First Week (essential onboarding material)
   - Background (helpful context and deeper understanding)
   - Reference (materials to consult when needed)

5. Limit the First Week category to no more than seven items.

6. If insufficient structured content is found, state this clearly and list the best available resources.

7. Output a structured onboarding path including:
   - Section headings (First Week, Background, Reference)
   - Document or page title (with link if available)
   - One-sentence explanation of why each item is relevant for this role

8. Do not modify or create content. Output guidance only.
